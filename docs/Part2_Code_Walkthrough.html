Lines 1–14: Header Comments

% ECG Signal: fs=360Hz, useful BW=0.5-100Hz, 11-bit/10mV
% Noise: baseline wander (<0.5Hz), 50Hz powerline, EMG (20-150Hz)
% Filter Specs:
%   HP:    cutoff=0.5Hz, PB ripple<=1dB, SB atten>=40dB
%   Notch: 50Hz center, PB ripple<=0.5dB, SB atten>=30dB
%   LP:    cutoff=100Hz, PB ripple<=1dB, SB atten>=40dB

These comments summarize the engineering specifications. PB ripple = how much the magnitude is allowed to deviate from 0 dB in the passband. SB atten = how far below 0 dB the filter must push frequencies in the stopband. These numbers come from clinical ECG standards (AHA guidelines).

Lines 24–26: Loading ECG Data

[sig100,~,~] = rdsamp('100',[],N_seg);

rdsamp: A function from the WFDB Toolbox. It reads the binary .dat files from the MIT-BIH database. It returns a matrix where each column is a different ECG lead.

sig100(:,1): Extracts column 1, which is the Modified Lead II (MLII) — the standard lead for arrhythmia detection.

t100 = (0:length(ecg100)-1)/fs: Creates a time vector in seconds.

Lines 28–32: FIR Filter Design (Hamming Window)

N_hp_fir = 500;
b_fir_hp = fir1(N_hp_fir, 0.5/(fs/2), 'high', hamming(N_hp_fir+1));
a_fir_hp = 1;

What fir1 does:

  1. Designs a windowed-sinc FIR filterFIR FilterFinite Impulse Response. A highly stable filter with linear phase (doesn't distort the heartbeat shape), but requires a lot of computing power. of order N (which means N+1 = 501 coefficients).
  2. 0.5/(fs/2) = 0.5/180 = 0.00278 — the normalized cutoff frequency.
  3. 'high' = high-pass filter (blocks below cutoff, passes above).
  4. hamming(501) = apply a 501-point Hamming window to suppress Gibbs ringing.

Why order 500?

The HP filter must transition from blocking at 0 Hz to passing at 0.5 Hz. That's a tiny normalized frequency (0.00278). The rule of thumb for fir1 with Hamming is: N ≈ 3.3 / (transition_width / fs). For a 0.5 Hz transition, we need a massive order. We use 500 as a practical compromise.

Why a_fir_hp = 1?

FIR filters have no denominator (no feedback). Setting a = 1 tells MATLAB there's no recursive part.

N_notch_fir = 500;
b_fir_notch = fir1(N_notch_fir, [48 52]/(fs/2), 'stop', hamming(N_notch_fir+1));

[48 52]/(fs/2): Two cutoff frequencies → bandstop (notch). Blocks 48–52 Hz, passes everything else.

N_lp_fir = 100;
b_fir_lp = fir1(N_lp_fir, 100/(fs/2), 'low', hamming(N_lp_fir+1));

Only order 100 because 100 Hz is a relatively high frequency (0.556 normalized) — the transition band is wide in absolute terms, so fewer taps suffice.

Lines 34–37: IIR Butterworth Design

[b_but_hp, a_but_hp] = butter(4, 0.5/(fs/2), 'high');

What butter does:

Designs an analog Butterworth prototype (maximally flat) and converts it to digital using the bilinear transform. It returns both b (numerator) and a (denominator) coefficients.

Why order 4?

Butterworth at order 4 gives ~80 dB/decade rolloff. This is steep enough for ECG. Compare: our FIR needed order 500 for the same cutoff. IIR's feedback makes it exponentially more efficient.

w0 = 50/(fs/2);
bw = w0/30;
[b_but_notch, a_but_notch] = iirnotch(w0, bw);

What iirnotch does:

Creates a 2nd-order IIR notch filter that places a deep null at exactly w0. bw = w0/30 defines the Q-factor (Quality factor = 30). Higher Q = narrower notch. Q=30 means the notch is only about 50 ± 0.83 Hz — surgically precise.

Lines 39–42: IIR Chebyshev Type II Design

[b_ch2_hp, a_ch2_hp] = cheby2(4, 40, 0.5/(fs/2), 'high');

Same process as butter, but with a Chebyshev Type II characteristic. 40 is the Rs = 40 dB — the minimum stopband attenuation. Noise in the stopband will be at least 40 dB (100×) below the passband. We use Chebyshev II because it gives a sharper transition than Butterworth at the same order, and keeps the passband perfectly flat.

Lines 51–68: Applying the Filter Cascades

FIR Cascade

% FIR cascade (Standard filtering)
ecg_fir = filter(b_fir_hp, 1, ecg);
ecg_fir = filter(b_fir_notch, 1, ecg_fir);
ecg_fir = filter(b_fir_lp, 1, ecg_fir);

We use standard filter(). Since FIR already has linear phase, it won't distort the heartbeat shape. Using filtfilt on FIR would double the order unnecessarily and waste computation.

IIR Butterworth Cascade

% Butterworth cascade (Zero-phase filtering)
ecg_but = filtfilt(b_but_hp, a_but_hp, ecg);
ecg_but = filtfilt(b_but_notch, a_but_notch, ecg_but);
ecg_but = filtfilt(b_but_lp, a_but_lp, ecg_but);

This performs Zero-Phase FilteringZero-Phase Filtering (filtfilt)Filters the data in the forward direction, then reverses the filtered sequence and runs it back through the filter. Lines 70–81: PSD and SNR Computation

nw = 1024;
[pxx_raw, f_psd] = pwelch(ecg, hamming(nw), nw/2, nw, fs);

pwelch divides the signal into segments of 1024 samples, overlaps them by 50%, applies a Hamming window, takes the FFT, squares it, and averages them all. This produces a smooth, statistically reliable PSD estimate unlike a raw, noisy FFT.

sb = f_psd>=0.5 & f_psd<=40;    % signal band
nb = (f_psd<0.5) | (f_psd>=49 & f_psd<=51) | (f_psd>100);  % noise bands

snr_raw = 10*log10(sum(pxx_raw(sb)) / (sum(pxx_raw(nb)) + eps));

Calculates the Signal-to-Noise Ratio. sb masks the frequencies where real ECG content lives. nb masks the noise frequencies. We sum the power in each band, divide them, and convert to decibels using 10*log10().

Lines 107–113: Frequency Response Plots

[H, f] = freqz(b, a, Nfft, fs);
plot(ax, f, 20*log10(abs(H)+eps));    % Magnitude
plot(ax, f, unwrap(angle(H))*180/pi); % Phase

freqz evaluates the filter's transfer function H at Nfft equally-spaced frequencies. We plot the magnitude in dB (20*log10(abs(H))) and the phase in degrees (angle(H)*180/pi). unwrap removes artificial ±π jumps.

Lines 118–131: Impulse and Step Response

imp = [1; zeros(imp_len-1,1)];  % δ[n]: a single spike
stp = ones(imp_len,1);           % u[n]: a constant "on" signal
h_i = filter(b, a, imp);         % impulse response
h_s = filter(b, a, stp);         % step response

We feed a single spike to get the impulse response, and a constant 1 to get the step response. We use filter here, not filtfilt, because we want to see the filter's true causal behavior in real time.

Lines 133–141: Pole-Zero Diagrams

zplane(b, a);

Plots the zeros (○) and poles (×) of the filter's transfer function in the complex z-plane. For stability, all poles MUST be strictly INSIDE the unit circle. For FIR, all poles are at z=0 (always stable).
